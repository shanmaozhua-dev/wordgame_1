# 00_总规则

本目录是地图制作协作的单一事实源。所有成员和所有 AI 在制作、修改、验收地图前，都必须先读这里的文件，而不是依赖聊天上下文猜规则。

## 必读顺序

1. `00_总规则.md`
2. `01_地图制作规范.md`
3. `02_事件与剧情规范.md`
4. `03_动画接入规范.md`
5. `04_命名与目录规范.md`
6. `05_验收清单.md`
7. `06_常见错误示例.md`
8. 当前关卡的 `level_manifest.json`
9. 当前关卡的 `handoff.md`

## 项目事实

- 源项目地图通过 `Global.map_transport(map_name)` 加载，真实路径规则是 `res://Scenes/Maps/<map_name>.tscn`。
- 源项目地图主结构通常是根 `Node`，子节点 `MainMap` 实例化自 `res://Scenes/MainMap.tscn`。
- `MainMap` 是 `TileMap`，脚本为 `Scripts/MainMap.gd`，默认 `bound = Rect2(0, 0, 32, 18)`。
- 地图、角色、事件的逻辑格子单位是 `60x60` 像素。
- 事件节点的 `position` 是格子左上角：`position = Vector2(grid_x * 60, grid_y * 60)`。
- 文字显示节点 `WordSprite` 在事件内部偏移 `Vector2(30, 30)`，也就是文字视觉中心在格子中心。
- 单字碰撞形状默认 `RectangleShape2D extents = Vector2(30, 30)`，中心在 `Area2D.position = Vector2(30, 30)`。
- `Event.text`、`Event.can_push`、`Event.can_delete`、`Event.can_split`、`Event.layer` 等属性是关卡交互的核心字段。

## 素材来源硬规则

- 所有动画、音效、贴图来源必须记录自 `D:\文字游戏`。
- 不允许为了赶进度自创动画、替代动画、临时贴图或临时音效。
- 当前仓库中的源代码可作为分析镜像使用，但交接文件里仍要记录对应的 `D:\文字游戏\...` 源路径。
- 如果源路径缺失，必须在 `handoff.md` 的风险里写明，不允许悄悄替换。

## 坐标规则

- 逻辑坐标：`Vector2(grid_x, grid_y)`，例如 `now_pos = Vector2(1, 6)`。
- 像素坐标：`Vector2(grid_x * 60, grid_y * 60)`，例如 `position = Vector2(60, 360)`。
- 事件锚点：格子左上角。
- 字节点视觉锚点：格子中心，即事件内部 `WordSprite.position = Vector2(30, 30)`。
- 地图文字、事件文字必须完整落在 60x60 格子体系内。多行或大字事件必须以 60 像素为步长对齐。

## 复刻等级

必须 1:1 复刻：

- 地图目录、地图名、章节顺序。
- 主线地图切换关系。
- 玩家出生点、门、触发器、关键事件点。
- 文字交互属性：可推、可删、可拆、可合、层级阻挡。
- 原项目已有动画、音效、贴图的来源与触发时机。
- 句子规则、拆字规则、合字规则。

允许阶段性简化：

- 非关键装饰事件。
- 不影响流程的细碎镜头表现。
- 尚未接入的远景或环境层效果。
- 复杂动画的内部轨道可以先保留源资源引用，后续再精修。

简化必须写进 `handoff.md` 的“未完成”和“风险”。

## 工作边界

- 地图制作只改当前负责地图及其必要资源引用。
- 不修改全局脚本行为，除非该改动已经被组内确认。
- 不重命名源地图、源事件名、源开关名，除非同时更新所有引用。
- 每次交付必须附带 `handoff.md` 和对应 `level_manifest.json`。

## 完成定义

一个地图只有同时满足以下条件，才能标记为完成：

- 可以从上一个地图进入。
- 可以完成本地图主流程。
- 可以切到下一个地图。
- 关键文字都在 60x60 格子内。
- 关键动画、音效、贴图来源可追溯。
- 没有缺贴图、缺脚本、缺音效报错。
- `level_manifest.json` 和 `handoff.md` 已更新。
