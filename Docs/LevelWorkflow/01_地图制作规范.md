# 01_地图制作规范

本文件规定每个地图场景如何创建、摆放、连接和验收。

## 基础场景结构

新地图应以源项目结构为基准：

```text
Node
└── MainMap  instance: res://Scenes/MainMap.tscn
    ├── Player，可选；没有时 MainMap 会自动实例化 Player
    ├── Event / TriggerEvent / BigEvent / 自定义事件
    ├── TopLayer / FixedMap
    ├── Interpreter
    └── GamePlayTimer
```

优先复制或参考 `Scenes/Maps/系統/map_template.tscn` 和相近章节地图，不要从空场景凭感觉搭。

## MainMap 必填字段

每张地图至少确认这些导出属性：

| 字段 | 说明 |
|---|---|
| `bound` | 地图逻辑边界，默认常见值 `Rect2(0, 0, 32, 18)` |
| `player_spawn_pos` | 玩家出生格坐标，使用逻辑坐标 |
| `has_bgm` | 是否有 BGM |
| `bgm_path` | BGM 资源路径 |
| `bgm_transition_type` | `fade`、`crossfade`、`sync` |
| `has_env` | 是否有环境音 |
| `env_path` | 环境音路径 |
| `floor_type` | 行走音效地面类型，如 `wood`、`dirt`、`stone` |
| `has_reverb` | 是否开启混响 |
| `reverb_mix_rate` | 混响比例 |
| `sentence_rules` | 句子规则 JSON |
| `split_list` | 拆字规则 |
| `merge_list` | 合字规则 |

## 地图尺寸

- 逻辑尺寸由 `MainMap.bound` 决定。
- 默认基础模板是 `32 x 18` 格，每格 `60 x 60` 像素。
- 地图像素尺寸公式：`bound.size * 60`。
- 摄像机和规则扫描都依赖 `bound`，不能只画 TileMap 而忘记设置 `bound`。

## TileSet 与通行

源码通行规则在 `BaseMap.can_pass(pos)`：

- TileMap 中 `cell_source_id > 0` 表示地面可通行。
- 空格或无 tile 通常不可通行。
- `MainMap.on_set_bound()` 会在 `bound` 范围内填 source id `1`。
- 固定层 `TopLayer/FixedMap` 也会参与阻挡判断。

制作时要保证：

- 玩家可走区域有 tile。
- 不可走区域无 tile 或被前景/固定事件阻挡。
- 地图边界外不能成为主流程必经路径。

## 坐标与摆放

事件节点摆放公式：

```gdscript
position = Vector2(grid_x * 60, grid_y * 60)
now_pos = Vector2(grid_x, grid_y)
```

示例：

```text
position = Vector2(60, 360)
now_pos = Vector2(1, 6)
```

文字视觉中心由事件内部的 `WordSprite.position = Vector2(30, 30)` 处理，不要把事件本体摆到格子中心。

## 玩家出生点

优先使用 `MainMap.player_spawn_pos`。

- 坐标是逻辑格坐标，不是像素坐标。
- 地图切换时如果命令传入 `spawn_pos`，会优先使用 `Global.player_status_for_map_change["spawn_pos"]`。
- 如果场景里没有 Player，`MainMap` 会自动实例化 `res://Scenes/Player/Player.tscn`。

## 门、触发器、事件点

常用节点：

| 节点 | 用途 |
|---|---|
| `Event.tscn` | 单格文字事件、门、NPC、可推可删文字 |
| `TriggerEvent.tscn` | 触发区域，通常半透明，属于 `trigger_event` 组 |
| `BigEvent.tscn` | 多字/大范围文字事件 |
| `WordSprite.tscn` | 仅文字绘制，不承担完整事件逻辑 |

常用触发方式：

| 值 | 枚举 | 说明 |
|---:|---|---|
| `0` | `AUTO` | 自动触发 |
| `1` | `PRESS` | 面前按确认键触发 |
| `2` | `TOUCH` | 玩家走到/碰到触发 |
| `3` | `LOOP` | 循环执行 |

门事件通常使用 `TOUCH` 或 `PRESS`，命令中接 `map_transport`。

## 层级与阻挡

`Event.layer` 来源：

| 值 | 枚举 | 用途 |
|---:|---|---|
| `0` | `BACK` | 背景层，通常不阻挡 |
| `1` | `MID` | 中层，默认可阻挡玩家/事件 |
| `2` | `FRONT` | 前景层，用于遮挡或视觉前景 |

默认 `Event` 的 `z_index = 10`，前景常用 `z_index = 20`，背景常用 `z_index = 0`。

## 地图制作流程

1. 复制相近源地图或 `map_template.tscn`。
2. 设置 `MainMap.bound`。
3. 设置 `player_spawn_pos`。
4. 铺可通行 Tile。
5. 摆放关键事件：门、触发器、剧情事件、可交互文字。
6. 填写事件 `text`、`now_pos`、`position`、`layer`、`can_push`、`can_delete`、`can_split`。
7. 填写事件 `commands`。
8. 接入 BGM、环境音、动画、音效。
9. 填写或更新 `level_manifest.json`。
10. 按 `05_验收清单.md` 验收。

## 每关必须留下的信息

- 源地图路径。
- 复刻目标地图路径。
- 地图边界。
- 玩家出生点。
- 本关入口和出口。
- 所有关键事件名称。
- 使用的动画、贴图、音效。
- 未完成项和风险。
