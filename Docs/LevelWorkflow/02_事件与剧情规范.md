# 02_事件与剧情规范

本文件规定事件节点、剧情命令、开关、场景切换和文字规则如何写。

## 事件节点类型

| 类型 | 源场景 | 用途 |
|---|---|---|
| 普通事件 | `res://Scenes/Events/Event.tscn` | 门、NPC、单字、可推可删可拆文字 |
| 触发事件 | `res://Scenes/Events/TriggerEvent.tscn` | 玩家碰到后触发剧情或区域逻辑 |
| 大字事件 | `res://Scenes/Events/BigEvent.tscn` | 多格文字、大块文本、特殊显示 |
| 角色事件 | `Poet.tscn`、`Princess.tscn`、`OldManEvent.tscn` 等 | 特殊角色表现 |

## Event 核心字段

| 字段 | 说明 |
|---|---|
| `name` | 事件名，也是命令 `target` 常用目标 |
| `position` | 像素坐标，格子左上角 |
| `now_pos` | 逻辑格坐标 |
| `text` | 显示文字 |
| `text_color` | 文字颜色 |
| `event_trigger_action` | 触发方式：`AUTO/PRESS/TOUCH/LOOP` |
| `layer` | 层级：`BACK/MID/FRONT` |
| `can_push` | 能否被推 |
| `can_delete` | 能否被 Backspace 删除 |
| `can_split` | 能否被 Tab 拆字 |
| `exist_condition` | 出现条件 |
| `commands` | 默认触发命令 |
| `loop_move_route` | 循环移动命令 |
| `copy_from_event` | 复制另一个事件的命令 |
| `need_be_saved` | 是否保存到存档 |

## 命令格式

事件命令写在 `commands` 字段，格式是：

```text
@[command_name] parameters
```

参数通常是 JSON 或字符串。示例：

```text
@[type] {"pos":[0,6],"texts":"房外是一條長廊。","is_dialog_end":true}
@[wait] 30
@[play_se] {"path":"res://Sounds/se/第一章 音效/SE_1_3_door_open.wav"}
@[map_transport] {
  "map": "第一章/03_房間",
  "need_to_save_map_status": false,
  "transition_type": "fade",
  "opacity": 1
}
```

JSON 内不能留下非法逗号、未闭合引号或不完整注释。源项目里有少量 `##` 注释写法，复刻时优先保持 JSON 干净。

| `transport_event` | 把事件/玩家传送到某格 |

## 地图切换格式

正确：

```text
@[map_transport] {
  "map": "第一章/03_房間",
  "need_to_save_map_status": false,
  "transition_type": "fade",
  "opacity": 1
}
```

注意：

- `map` 不写 `res://Scenes/Maps/`。
- `map` 不写 `.tscn`。
- `transition_type` 常见值：`fade`、`white_fade`、`fast_fade`、`whirl`、`none`。
- 地图文件名必须与 `Scenes/Maps` 下真实路径一致。

## 对话与文本

`type` 常见参数：

```json
{
  "pos": [0, 6],
  "texts": "房外是一條長廊。",
  "is_dialog_end": true
}
```

规则：

- `pos` 是格坐标，不是像素。
- 文本中的 `[事件名]` 通常表示引用或强调某个文字事件。
- `&`、`|` 等节奏符号要照源项目保留。
- 不确定含义时不要改写原文。

## 特殊命令段

`Event.raw_code_to_json_command()` 支持按动作抽取特殊段：

```text
@[backspace_command]
  ...
@[end_backspace_command]

@[split_command]
  ...
@[end_split_command]
```

规则：

- 默认触发只读取普通段。
- Backspace 触发读取 `backspace_command` 段。
- Tab 拆字触发读取 `split_command` 段。

## 推、删、拆、合

玩家逻辑在 `Player.gd`：

- 推：玩家拥有 `has_push_power`，正面事件 `layer == MID` 且 `can_push == true`。
- 拉：按 Alt 锁定可推事件，随后反方向移动。
- 删：玩家拥有 `has_backspace_power`，目标事件 `can_delete == true`。
- 拆：玩家拥有 `has_split_power`，目标事件 `can_split == true`，且 `MainMap.split_list` 中存在规则。
- 合：玩家推动前方事件时，如果前方下一格事件与当前事件满足 `MainMap.merge_list`，合成新事件。

## 句子规则

`MainMap.sentence_rules` 是 JSON 字符串。规则会扫描地图横向和纵向文本。

示例结构：

```json
[
  {
    "text": "天氣很好",
    "switch": "ch_example_weather_good",
    "has_hint": false,
    "has_animation": true,
    "progress": 0,
    "level": 5
  }
]
```

规则：

- `text` 是要匹配的句子。
- `＊` 可作为通配符。
- `except` 可限制通配符不能匹配的字符。
- 触发成功会设置 `switch`。
- 验收时必须确认句子能触发，也能在拆掉后恢复/取消。

## 第一关流程拆段建议

以第一章为例，按文件和流程拆：

1. `第一章/00_第一章字卡`：章节字卡。
2. `第一章/01_開頭`：开场剧情。
3. `第一章/02_長廊`：长廊、门、第一段可交互流程。
4. `第一章/03_房間`：房间主交互。
5. 后续按地图文件名继续拆。

每段都要独立记录入口、出口、关键事件、已接动画和未接资源。
