# 06_常见错误示例

这里收集地图制作中最容易拖慢进度的问题。遇到类似情况，先对照本文件排查。

## 错误 1：把事件摆在格子中心

错误：

```text
now_pos = Vector2(1, 6)
position = Vector2(90, 390)
```

正确：

```text
now_pos = Vector2(1, 6)
position = Vector2(60, 360)
```

说明：事件本体是格子左上角，文字中心由 `WordSprite.position = Vector2(30, 30)` 处理。

## 错误 2：把 `map_transport` 写成完整路径

错误：

```text
@[map_transport] {"map":"res://Scenes/Maps/第一章/03_房間.tscn"}
```

正确：

```text
@[map_transport] {"map":"第一章/03_房間"}
```

说明：源码会自动拼 `res://Scenes/Maps/` 和 `.tscn`。

## 错误 3：自创动画占位

错误：

```text
使用自制 door_open_placeholder.tscn
```

正确：

```text
source: D:/文字游戏/Sprites/door/door_open.png
target: res://Sprites/door/door_open.png
```

说明：动画必须能追溯到 `D:\文字游戏`。

## 错误 4：改了事件名但没改 target

错误：

```text
[node name="Door"]
@[play_sprite_animation] {"target":"門"}
```

正确：

```text
[node name="門"]
@[play_sprite_animation] {"target":"門"}
```

说明：事件名就是命令寻址键。除非全图同步更新，否则不要改。

## 错误 5：忘记 `bound`

表现：

- 玩家走到不该去的位置。
- 句子规则扫描不到文本。
- 摄像机边界异常。

处理：

- 设置 `MainMap.bound`。
- 确认可走 Tile 与 `bound` 一致。

## 错误 6：把 60 像素规则和复刻测试里的 72 像素混用

源项目地图是 `60x60` 格子。复刻测试场景如果使用其他 cell size，只能用于模块测试，不能写进正式地图规范。

## 错误 7：JSON 命令不可解析

错误：

```text
@[type] {"pos":[0,6],"texts":"文本",}
```

正确：

```text
@[type] {"pos":[0,6],"texts":"文本"}
```

处理：

- 优先使用标准 JSON。
- 大段命令复制后先检查引号、逗号和括号。

## 错误 8：把 `can_push/can_delete/can_split` 当成玩家能力

事件字段：

```text
can_push = true
can_delete = true
can_split = true
```

玩家能力：

```text
Global.player_status.has_push_power
Global.player_status.has_backspace_power
Global.player_status.has_split_power
```

两者都要满足，交互才会发生。

## 错误 9：忘记 `need_be_saved = false`

自动剧情、遮罩、一次性触发器等源项目常会设置：

```text
need_be_saved = false
```

如果误保存，可能导致读档后事件状态异常。

## 错误 10：层级写错导致阻挡异常

默认中层 `MID` 会参与阻挡。背景遮罩、纯视觉节点通常应该是 `BACK` 或设置为不阻挡。

检查项：

- `layer`
- `z_index`
- 是否属于 `events` 组
- 是否有碰撞形状

## 错误 11：缺资源但没有记录

错误做法：

- 控制台报缺贴图，交接不写。
- 先用别的音效顶替。

正确做法：

- 在 `handoff.md` 的风险写清楚。
- 在 `level_manifest.json` 的 `missing_resources` 写清楚。

## 错误 12：只看画面不跑流程

地图看起来对，不代表能交付。必须至少跑：

1. 上一地图进入。
2. 本地图关键事件。
3. 本地图出口。
4. 下一地图加载。
