# 文字机制复刻说明

第三章的核心是“文字既是叙事，也是可操作物体”。复刻时不要把文字只当作台词渲染，它们必须能被玩家推动、删除、拆分、合成，并参与规则判定。

## 1. 调查文字

调查是最普通的交互。玩家面对 `event_trigger_action = PRESS` 的 Event 并按确认键，会执行该 Event 的 `commands`。

常见用途：

- 触发对话。
- 生成新的文字。
- 改变开关。
- 开启能力。
- 切换地图。
- 播放演出。

第三章很多 NPC、路牌、门、机关都属于调查交互。

## 2. 推字

条件：

```text
player.has_push_power = true
event.can_push = true
目标格可通行
```

效果：

- 文字 Event 沿玩家方向移动一格。
- 移动结束后调用地图规则检查。
- 如果被推成指定句子，打开对应开关。

推字是 `04_手套教學` 后半段、`13_添譜來堂_方塊`、`15_添譜來堂_拳頭` 的核心。

## 3. 退格/删除

条件：

```text
player.has_backspace_power = true
event.can_delete = true
```

默认效果：

- Event 播放被删除的动画。
- 该文字从地图移除。
- 地图规则重新检查。

特殊情况：

- 如果 Event 写了 `@[backspace_command]`，退格时不直接删除，而是先执行特殊指令。
- `@[end_backspace_command]` 用于删除动画/流程结束后的补充逻辑。

复刻时建议给 Event 增加：

```text
onBackspaceCommand?: Command[]
onBackspaceEndCommand?: Command[]
```

## 4. 拆字

条件：

```text
player.has_split_power = true
event.can_split = true
地图 split_list 中存在该字的拆分规则
```

效果：

- 原文字播放拆分动画。
- 根据拆分表生成多个新文字。
- 新文字可继承或获得 `can_push`、`can_split` 等属性。

第三章中拆字不是最早需要复刻的重点，但如果要还原后续复杂文字机关，需要保留这个扩展点。

## 5. 合字

地图有 `merge_list`。当多个文字满足合并关系时，可生成合并后的文字。

复刻早期关卡可以先不实现复杂合字，只保留数据结构：

```text
mergeRules: [
  { from: ["木", "木"], to: "林" }
]
```

## 6. 成句规则

成句规则是第三章谜题的胜利判定核心。

玩家推动或删除文字后，地图会扫描横向和纵向文字。如果连续文字匹配规则中的 `text`，就打开 `switch`。

复刻模型：

```json
{
  "text": "不猶豫地向前踏出了步伐。勇者的覺悟不言自明。",
  "switch": "some_switch",
  "has_animation": true
}
```

匹配成功后通常会：

- 播放“成句合法”动画。
- 打开出口、剧情或下一段事件。
- 有时记录匹配位置供后续演出使用。

## 7. Typewriter 生成可交互文字

第三章很多可交互文字不是手动摆在场景里的，而是由 `type` 指令生成。例如：

```text
不{can_push:true,can_delete:true}
```

这表示生成文字 `不`，且它可推、可删。

复刻 Typewriter 时至少要支持：

```text
逐字生成
换行 &
等待 |
空格 ＿
内联属性 {...}
复用已有事件 [name]
移动玩家 Ｍ
```

## 8. 例子：04_手套教學

这是最适合作为第一关复刻样本的地图：

- 文件：`D:\WordGame\Scenes\Maps\第三章\04_手套教學.tscn`
- 机制：退格教学、推字教学、成句规则、能力解锁、地图切换。

大致流程：

1. 关卡开启退格能力。
2. 文字生成一段句子，里面有可删的 `不`。
3. 玩家删除 `不`，改变句意并推动剧情。
4. 后续生成可推的 `不`。
5. 玩家把 `不` 推到正确位置，形成合法句。
6. 成句规则成功，解锁推字能力或进入下一张地图。

关键原始线索：

- `set_backspace_power`：开启退格能力。
- `不{can_push:true,can_delete:true}`：生成可推/可删的 `不`。
- `append_sentence_rule`：添加需要组成的句子规则。
- `set_push_power`：开启推字能力。
- `map_transport`：通向 `第三章/05_宿命小徑`。

这关的复刻重点不是每一帧演出，而是把“删除一个否定字改变句意”和“推动否定字组成新句子”的体验做对。

## 9. 例子：13_添譜來堂_方塊

- 文件：`D:\WordGame\Scenes\Maps\第三章\13_添譜來堂_方塊.tscn`
- 机制：大量可推文字、句子条带、成句规则。
- 适合验证推字、碰撞、横纵向规则扫描。

复刻时先查：

- `D:\WordGame\第三章可交互文字清单.md` 中 `13_添譜來堂_方塊` 章节。
- 原 `.tscn` 里的 `commands`、`can_push`、`append_sentence_rule`。

这关要保证同一关卡不同位置出现的同一文字，如果交互方式一致，可以复用同一个文字配置，但需要在关卡数据里保留数量和坐标。

## 10. 例子：15_添譜來堂_拳頭

- 文件：`D:\WordGame\Scenes\Maps\第三章\15_添譜來堂_拳頭.tscn`
- 机制：推字、删字、战斗/手势演出、规则判断。
- 常见交互文字包括 `贏`、`不`、`二`、`讚`、`一`、`零`、`好`、`劍` 等。

复刻时注意：

- `劍` 相关事件会影响退格/删除能力或战斗节奏。
- 数字和评价词可能需要被推到特定结构里。
- 这关比 `13_添譜來堂_方塊` 更依赖演出和状态切换，建议在方块关之后再做。
