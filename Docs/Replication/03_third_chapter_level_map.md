# 第三章关卡流与复刻重点

本文按第三章地图顺序整理关卡用途、核心机制和复刻优先级。具体可交互文字明细请查：

- `D:\WordGame\第三章可交互文字清单.md`

## 1. 第三章主流程

大致地图流：

```text
00_第三章字卡
-> 00_木林森過場
-> 01_開場
-> 02_1_幻覺
-> 02_2_腳印
-> 02_勇者之村
-> 03_酒館
-> 04_手套教學
-> 05_宿命小徑
-> 06_深巷廢墟_長巷
-> 07_深巷廢墟_廢墟
-> 08_深巷廢墟_腳印
-> 09_人骨教堂外
-> 10_人骨教堂
-> 11_添譜來堂_開場
-> 12_添譜來堂_生肖
-> 13_添譜來堂_方塊
-> 14_添譜來堂_拳頭轉場
-> 15_添譜來堂_拳頭
-> 16_添譜來堂_尾聲
-> 17_第三章結束
```

并非每张地图都是谜题关。有些是过场、剧情、教学或转场。

## 2. 推荐复刻顺序

### A. `04_手套教學`

复刻优先级：最高。

原因：

- 同时包含退格、推字、成句、能力开启、地图切换。
- 是第三章文字机制的最小完整样本。
- 适合先验证系统架构。

核心体验：

- 玩家获得退格能力。
- 删除句子中的 `不`。
- 获得或使用推字能力。
- 推动 `不` 组成正确句子。
- 成句成功后进入 `05_宿命小徑`。

需要实现：

- Typewriter 生成文字。
- `can_delete`。
- `can_push`。
- 成句规则。
- 能力开关。

### B. `13_添譜來堂_方塊`

复刻优先级：高。

原因：

- 是比较完整的推字成句谜题。
- 可用于测试多文字、多位置、横纵向扫描。

核心体验：

- 场景中有大量可推文字。
- 玩家调整文字位置，使其组成合法句。
- 合法句触发开关和后续演出。

需要实现：

- 稳定的推字碰撞。
- 多个同字事件。
- 成句规则。
- 关卡内状态变化。

### C. `15_添譜來堂_拳頭`

复刻优先级：高，但建议排在 `13_添譜來堂_方塊` 后。

原因：

- 机制组合更复杂。
- 既有文字谜题，也有战斗/手势/剧情演出。

核心体验：

- 操作 `贏`、`不`、`一`、`二`、`零`、`好`、`讚`、`劍` 等文字。
- 通过推、删、规则触发改变局势。
- 事件演出和状态切换较多。

需要实现：

- 推字。
- 退格。
- 事件链。
- 战斗或手势演出状态。
- 更细的开关变量。

### D. `02_勇者之村` 与 `03_酒館`

复刻优先级：中。

原因：

- 更偏探索和叙事。
- 适合补世界观、NPC 对话、章节节奏。

核心体验：

- 调查 NPC、物件、路标。
- 触发对话和地图推进。
- 为 `04_手套教學` 做叙事铺垫。

需要实现：

- 调查事件。
- 对话/打字演出。
- 地图切换。

### E. `06_深巷廢墟_*` 系列

复刻优先级：中。

原因：

- 是手套教学后的过渡区域。
- 可以用来测试推字能力在探索地图中的使用。

核心体验：

- 在长巷、废墟、脚印场景中继续推进。
- 有部分可交互文字和剧情事件。

需要实现：

- 基础调查。
- 推字。
- 地图跳转。

### F. `10_人骨教堂` 与 `12_添譜來堂_生肖`

复刻优先级：中低。

原因：

- 更依赖章节叙事和具体演出。
- 如果只复刻机制样本，可以后做。

核心体验：

- 调查和剧情推进。
- 特定文字机关。
- 为后续添譜來堂谜题铺垫。

## 3. 每关数据整理模板

复刻某一关时，建议按下面格式建关卡数据：

```yaml
id: "第三章/04_手套教學"
displayName: "手套教學"
grid:
  cellSize: 60
playerStart:
  x: 0
  y: 0
  direction: "down"
powers:
  push: false
  backspace: true
  split: false
events:
  - id: "word_bu_01"
    text: "不"
    x: 10
    y: 8
    canPush: true
    canDelete: true
    trigger: "none"
rules:
  - text: "不猶豫地向前踏出了步伐。勇者的覺悟不言自明。"
    switch: "rule_clear"
    animation: true
onClear:
  - command: "map_transport"
    target: "第三章/05_宿命小徑"
```

实际坐标和完整事件需从原 `.tscn` 与交互文字清单补齐。

## 4. 数据来源关系

复刻时请把信息分成三层：

- 系统层：从 `Scripts/*.gd` 读，决定机制怎么工作。
- 关卡层：从 `Scenes/Maps/第三章/*.tscn` 读，决定每关有什么事件、坐标、指令。
- 摘要层：从 `第三章可交互文字清单.md` 读，快速定位玩家能操作的文字。

不要只依赖摘要层。摘要适合导航，但具体复刻坐标、演出顺序和开关名仍需要回源文件确认。
