# WordGame 关卡复刻资料包

这组文档用于把原项目里的关键信息交接给另一个 Codex 或开发者，使其能够理解游戏机制并复刻第三章的部分关卡。

建议阅读顺序：

1. `README.md`：先建立全局视角，知道项目里信息藏在哪里。
2. `01_core_systems.md`：理解事件、玩家、指令解释器、打字机、规则判定等核心系统。
3. `02_word_mechanics.md`：理解推字、退格、拆字、合字、成句规则这些复刻重点。
4. `03_third_chapter_level_map.md`：按第三章关卡流查看每关目标、交互文字和胜利条件。
5. `04_data_extraction_guide.md`：需要补细节时，按这里的方法回到原 `.tscn` / `.gd` 文件查证。

第三章全部“玩家可交互文字”的明细已经整理在仓库根目录：

- `第三章可交互文字清单.md`

这份清单适合查具体文字、数量、交互方式和大致效果。本资料包则负责解释这些信息如何组成可复刻的关卡。

## 原项目基本信息

- 项目类型：Godot 4 项目。
- 主场景：`res://Scenes/UI/StartMenu.tscn`
- 地图目录：`Scenes/Maps`
- 第三章地图目录：`Scenes/Maps/第三章`
- 核心脚本目录：`Scripts`

主要全局单例：

- `Global`：章节进入、地图切换、存档、开关、变量、玩家状态。
- `InputSystem`：输入状态。
- `Typewriter`：把文本逐字生成到地图上。
- `UI`：UI 显示。
- `Sound`：音效和音乐。
- `EscMenu`：菜单和文档/碎碎念等数据入口。

常见输入：

- 移动：方向键或 WASD。
- 确认/调查：空格。
- 取消/菜单：Esc。
- 退格能力：Backspace 或 Delete。
- 其他能力/组合键：Ctrl、Z、Tab、Alt、Shift。

## 复刻时最重要的抽象

这个游戏的关卡不是传统“静态文字说明”，而是由以下对象共同构成：

- 地图格子：文字和玩家都在网格上移动，原项目以 60 像素为一格。
- Event 文字实体：每个可见文字基本都是一个事件节点，可以拥有文本、颜色、碰撞、可推、可删、可拆等属性。
- 事件指令：每个 Event 可绑定一段 `commands`，玩家调查、触碰、自动触发、循环触发时执行。
- 打字机：关卡大量文字不是预先摆放，而是运行时由 `Typewriter` 从字符串生成。
- 成句规则：玩家把文字推成指定句子后，地图会设置开关并播放合法句动画。
- 开关/变量：控制剧情推进、机关状态、是否已经触发过某事件。

复刻部分关卡时，不必一开始完整复刻 Godot 架构，但必须保留这些玩法语义。否则第三章的核心谜题无法成立。

## 推荐复刻优先级

1. 先复刻 `04_手套教學`：它集中展示退格、推字、成句、能力解锁，是机制最好的最小样本。
2. 再复刻 `13_添譜來堂_方塊`：它是比较完整的推字成句谜题。
3. 接着复刻 `15_添譜來堂_拳頭`：它把推字、删字、战斗演出和规则判断混合起来。
4. 如果需要叙事流程，再补 `02_勇者之村`、`03_酒館`、`10_人骨教堂` 等探索/剧情关。

## 交付给另一个 Codex 的最短说明

如果只给另一个 Codex 一段话，可以这样说：

> 从仓库根目录开始，先阅读 `Docs/Replication/README.md`，再看 `Docs/Replication/01_core_systems.md` 和 `Docs/Replication/02_word_mechanics.md`。第三章所有玩家可交互文字在 `第三章可交互文字清单.md`。复刻时以 `04_手套教學`、`13_添譜來堂_方塊`、`15_添譜來堂_拳頭` 为优先样本，保留网格文字实体、推字/退格/拆字、事件指令、Typewriter 生成文字、成句规则和开关变量机制。
