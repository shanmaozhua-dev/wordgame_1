# 第四章文字交互复刻规格

> Companion JSON: `第四章文字交互复刻数据.json`. This is the optimized handoff spec for another Codex; source scenes are untouched.

## 给另一个 Codex 的读取规则

- Read the JSON first; this Markdown is the implementation guide.
- `items` are initial visible map text from `text` or `big_text`.
- `runtime_items` are generated by `@[type]` / `@[type_parallel]`; do not place them at scene start.
- `invisible_triggers` are events without visible text; build them as trigger/script objects.
- `dynamic_can_hints` are source-line hints where scripts set or test `can_push/delete/split`.
- Node name is not display text; always trust `text` / `big_text`.
- In `big_text`, `＿` means an empty cell.

## 坐标与 32 x 18 页面规则

- Source coordinate: `now_pos = Vector2(x, y)`.
- Page size: `32 x 18`.
- `page_x = floor(x / 32)`, `page_y = floor(y / 18)`.
- `local_x = x mod 32`, `local_y = y mod 18`; use non-negative modulo for negative positions.

## 最优复刻顺序

1. Recreate `items` first: position, size, and push/delete/split/press/touch.
2. Recreate `invisible_triggers`: transports, switches, ability grants, conditional story flow.
3. Wire `runtime_items` into the typewriter/story system.
4. Review `dynamic_can_hints` for chapter-specific scripted mechanics.
5. Run scene-by-scene verification against the source `.tscn` behavior.
