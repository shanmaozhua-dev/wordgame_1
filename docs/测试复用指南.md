# 测试关卡复用指南

这份文档说明如何复用当前测试框架，继续把截图、文字说明和解包源码转成可试玩、可测试的关卡。

## 复用目标

每个新测试关卡都应该同时具备三件事：

- 可肉眼试玩：打开 `Main.tscn` 后可以直接操纵 `我`。
- 可自动验证：把关键玩法写进 `tests/test_gameplay_core.gd` 或新的测试脚本。
- 可交接扩展：地图、交互、拆合字、造句规则都写成清楚的数据，而不是散落在场景节点里。

## 当前可复用模块

- `scripts/grid_world.gd`：唯一玩法真相来源。移动、碰撞、推、拉、删、拆、合、造句提示都在这里。
- `scripts/word_entity.gd`：每个文字实体的数据结构。
- `scripts/level_loader.gd`：当前测试关卡数据。后续可以拆成多个 `build_xxx_level()`；勇者试炼入口是 `build_hero_trial_fist_level()`。
- `scripts/main.gd`：把 `GridWorld` 渲染成可试玩画面。
- `scripts/demo_runner.gd`：F5 自动演示路线。
- `tests/test_gameplay_core.gd`：核心玩法回归测试。
- `Fonts/Zpix.ttf` 和 `Fonts/Zpix.tres`：原版风格字体资源。

## 新增关卡的推荐流程

1. 截图原版画面，标出玩家起点、可互动字、可推字、可删字、可拆字和目标规则。
2. 优先从解包 `.tscn` 里抽 `now_pos` 和 `big_text`，不要靠肉眼估墙。
3. 在 `scripts/level_loader.gd` 里新增一个构建函数，例如 `build_room_level()`。
4. 用 `rows` 或 `map_text_lines` 写地图草稿，空格/`＿` 表示空地，中文字符表示地图文字。
5. 在 `entities` 里给特殊字补属性。
6. 在 `split_rules`、`merge_rules`、`sentence_rules` 里补关卡规则。
7. 在 `tests/` 里写至少一个自动测试，覆盖本关最关键的解法路径。
8. 手动运行主场景，再跑 headless 测试。

提取原版 `big_text` 坐标的命令：

```powershell
python .\tools\extract_tscn_big_text.py "E:\文字游戏\文字遊戲.7z\文字游戏源码\Scenes\Maps\第三章\15_添譜來堂_拳頭.tscn" "零的手勢"
```

输出格式是 `字 x y`。例如本关已确认 `贏` 的源码 `now_pos` 是 `(1,2)`，也就是第三行第二列；`零的手勢` 的手掌墙来自 `now_pos=(7,0)` 加上 `big_text` 里每个 `掌` 的相对位置。

## 地图数据模板

```gdscript
static func build_room_level() -> Dictionary:
	return {
		"rows": [
			"墙墙墙墙墙",
			"墙我 石 戈墙",
			"墙  又  墙",
			"墙墙墙墙墙"
		],
		"player_start": Vector2i(1, 1),
		"screen_size": Vector2i(32, 18),
		"entities": {
			"石": {"pushable": true, "solid": true},
			"又": {"pushable": true, "solid": true},
			"戈": {"pushable": true, "solid": true},
			"手表": {
				"solid": true,
				"interact_text": "手表可以查看人类世界的时间"
			}
		},
		"split_rules": {"戏": ["又", "戈"]},
		"merge_rules": {"又+戈": "戏", "戈+又": "戏"},
		"sentence_rules": {
			"天气": {
				"message": "已识别",
				"caption_pos": Vector2i(16, 9)
			}
		}
	}
```

## 规则约定

- 一屏固定按 `32 x 18` 个文字格组织。先定格子坐标，再调渲染像素。
- 如果出现字与字视觉重叠，优先检查 `Fonts/Zpix.tres` 是否锁死 `fixed_size`，再检查 `scripts/main.gd` 的字号比例。
- 所有实体默认 `solid=true`，玩家不能和文字重叠。
- `pushable=true` 的字可以被推。
- Alt 拉字时，玩家只能朝远离被拉字的方向走。
- 推一个字撞上另一个字时，会先查 `merge_rules`；可合并则自动合字，不可合并才阻挡。
- `interact_text` 和 `sentence_rules.message` 都会生成地图提示文字。
- 提示文字默认不消失、有碰撞；重复触发同一句提示不会生成重复实体。
- 如果后续要让提示字可推、可删、可互动，把提示配置扩展到 `caption_pushable`、`caption_deletable`、`caption_interact_text`。

## 自动测试模板

```gdscript
func test_push_into_merge_target_auto_merges() -> void:
	var world = make_world()
	var part_left = world.find_first_entity_by_text("又")
	var part_right = world.find_first_entity_by_text("戈")
	world.move_entity_to(part_left.id, Vector2i(8, 4))
	world.move_entity_to(part_right.id, Vector2i(9, 4))
	world.player_pos = Vector2i(7, 4)

	var pushed = world.try_move_player(Vector2i(1, 0))

	assert_true(pushed.success, "pushing a mergeable part into another part succeeds")
	assert_equal(world.get_entity_at(Vector2i(9, 4)).text, "戏", "merged word appears")
```

## 验证命令

优先运行一键验证：

```powershell

```

这个脚本会完成：

- Godot 资源导入。
- 核心玩法测试。
- 字体资源测试。
- 主场景启动检查。
- 启动真实 Godot 窗口并自动截图。

## AI 自动验收要求

后续让 AI 改玩法或画面时，不应该只让人肉眼检查。每次改动至少执行：

1. 新增或更新对应自动测试。
2. 跑 `.\tools\run_all_tests.ps1`。
3. 查看 `test-output/main-scene-smoke.png`，确认抓到的是当前测试项目窗口。
4. 在回复里说明验证命令和截图路径。

如果截图脚本抓不到本次启动的 Godot 窗口，脚本必须失败，不能退化成截整个屏幕。

## 给 Codex 的关卡输入格式

后续让 Codex 继续复刻关卡时，建议这样给信息：

```text
目标：复刻第 X 关的测试版。
截图：附图。
玩家起点：第 A 行第 B 列。
地图文字：逐行写出或让 Codex 根据截图草拟。
特殊字：
- 石：可推
- 删：可删
- 手表：空格互动，提示“……”
规则：
- 戏 -> 又 + 戈
- 又 + 戈 -> 戏
- 天气：识别后生成“已识别”
验收：
- 手动能走通。
- 自动测试覆盖关键解法。
```

## 字体说明

当前测试画面使用 `Fonts/Zpix.ttf`，由本机参考工程 `E:\文字游戏\文字遊戲.7z\文字游戏源码\Fonts\Zpix.ttf` 复制而来，用于贴近原版视觉。推送公开 GitHub 前，建议再确认字体授权文件；如果仓库需要规避素材风险，可以替换成明确可分发的开源 CJK 像素字体，并保持 `Fonts/Zpix.tres` 的引用接口不变。
